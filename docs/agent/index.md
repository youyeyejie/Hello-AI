# Agent 总览

Agent 不是“会说话的模型”，而是“能拆任务、会调用工具、会继续推进”的系统。

## 它和普通聊天有什么不同

普通聊天通常是一次输入、一次输出。

Agent 更像一个循环：

1. 先理解任务
2. 再拆分步骤
3. 再调用工具
4. 再根据结果继续

## 先记住一句话

Agent 不是魔法，它只是把“规划、调用、观察、再规划”串起来了。

## 它为什么会失败

- 任务拆得太大
- 工具定义不清楚
- 没有停止条件
- 结果校验不够

## 一个最小 Agent 长什么样

- 目标输入
- 任务拆解
- 工具调用
- 结果观察
- 再次决策

## 典型场景

- 文件整理
- 信息检索
- 简单自动化
- 多步任务编排

## 学完这一章的目标

你应该知道：

- Agent 和工作流的区别
- 为什么 Agent 要循环
- 为什么 Agent 也需要限制和边界
