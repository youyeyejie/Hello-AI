# Lab 1: Prompt 改写

## 背景

你在工作里经常会遇到一种情况：需求说得不够清楚，模型输出就会飘。

这一题要训练的，就是把一句模糊话改成一个可执行 Prompt。

## 目标

把“我想了解 RAG”改写成一个能稳定产出结构化回答的 Prompt。

## 输入材料

原始需求：

```text
我想了解 RAG。
```

## 提示与约束

改写后的 Prompt 需要包含：

- 角色
- 任务
- 输出格式
- 读者定位
- 一个常见误区

## 参考答案

```text
你是面向中文小白的技术助理。
请用通俗中文解释 RAG 是什么。
要求：
1. 先给一句话定义
2. 再说明为什么需要 RAG
3. 再给一个最小例子
4. 最后列出 1 个常见误区
5. 输出分成 4 段，每段不超过 120 字
```

## 扩展思考

- 如果对象是产品经理，Prompt 要怎么改
- 如果对象是工程师，Prompt 要怎么改
- 如果需要 JSON 输出，Prompt 要怎么改

## 本地复现方式

你可以拿任意聊天模型做测试：

1. 先发原始需求
2. 再发改写后的 Prompt
3. 比较两次输出的稳定性和结构化程度
