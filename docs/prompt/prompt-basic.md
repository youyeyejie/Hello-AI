# Prompt 基础

Prompt 就是你给模型的输入设计。

它不是“魔法咒语”，而是一种把需求讲清楚的方式。

## 一个好 Prompt 通常包含什么

- 角色：你希望模型扮演谁
- 任务：你要它做什么
- 约束：不能做什么、必须满足什么
- 输出格式：你希望结果长什么样
- 示例：你希望它学着什么样子回答

## 最小模板

```text
你是一个中文技术助理。
请用三点解释 RAG 是什么。
要求：先给定义，再给例子，最后列出一个常见误区。
```

这个模板已经包含了最基本的可用信息：

- 角色明确
- 任务明确
- 约束明确
- 输出结构明确

## 更稳定一点的写法

```text
你是面向小白的技术助理。
请解释 RAG。
要求：
1. 用通俗中文
2. 不要使用英文缩写堆砌
3. 先说是什么，再说为什么需要，最后说一个最小例子
4. 输出为三段，每段不超过 120 字
```

## 常见问题

- 需求太散，一次塞太多任务
- 没说清读者是谁
- 没说清输出格式
- 没说清“好”的标准

## 调整思路

如果结果不好，通常不是先换模型，而是先改 Prompt。

常见顺序是：

1. 先缩小任务
2. 再补充约束
3. 再加示例
4. 最后再考虑换模型
